# Visual Design Skill

## Cuándo usar esta skill
Antes de crear o modificar cualquier archivo de interfaz: HTML, CSS, componentes, layouts.

## Principio fundamental
No cambies colores, fuentes ni espaciado a menos que el usuario lo pida explícitamente.
Si el usuario no define una paleta, pregunta antes de asumir.

## Sistema visual base
- Paleta: respetar los valores hex ya definidos en el proyecto. No introducir nuevos.
- Tipografía: usar las fuentes ya cargadas. No agregar Google Fonts sin aprobación.
- Espaciado: consistente con el sistema existente (Bootstrap 3 en este proyecto).

## Decisiones de diseño
- La estructura comunica jerarquía: titulares > subtítulos > cuerpo, sin saltar niveles.
- Numeración (01/02/03) solo si el contenido es una secuencia lógica real.
- Animaciones solo si aportan orientación al usuario, no decoración.
- Cada elemento cumple una sola función: una etiqueta identifica, un botón ejecuta.

## Redacción en interfaz
- Voz activa: "Guardar cambios", no "Enviar formulario".
- Errores: explican qué pasó y cómo resolverlo, sin disculpas.
- Pantallas vacías: invitan a actuar, no describen el vacío.

## Restricción clave
Antes de salir, quita un accesorio. Si algo no cumple una función clara, elimínalo.
